# Slither configuration rationale

Companion to `.slither.config.json`. JSON cannot carry comments, so the *why* of each setting lives here.

## `filter_paths`: `"lib|test|script"`

Skip third-party libraries (OpenZeppelin, forge-std), our own Foundry tests, and our deployment scripts. Only the contracts in `src/` are production code; the other directories use cheatcodes or vendor code that confuse static analysis without surfacing real findings.

## `exclude_dependencies`: `true`

Slither classifies imported packages as dependencies. Suppresses upstream OpenZeppelin / forge-std findings that are not ours to fix.

## `detectors_to_exclude`: `"timestamp,solc-version,unindexed-event-address"`

- **timestamp** — `NoethrionAttester.finalizeBatch()` uses `block.timestamp` for the challenge-window comparison. Per ADR-006 open questions, validator clock skew of ±15 seconds is operationally irrelevant given hour-scale challenge windows. The Solidity lint flags the same line — already accepted there.
- **solc-version** — We pin `0.8.24` deliberately in `foundry.toml` for the `cancun` EVM target. Slither's generic "use a recent version" nag is not actionable for our deployment surface.
- **unindexed-event-address** — Slither flags `Pausable.Paused(address)` and `Pausable.Unpaused(address)` from OpenZeppelin's library. Those events live in `lib/openzeppelin-contracts/` and we cannot modify them. The `filter_paths` and `exclude_dependencies` config keys do not suppress events that originate in inherited contracts. Detector excluded globally; re-evaluate when OZ upstream addresses the warning or when Slither offers per-contract event filtering. Our own events (`BatchProposed`, `BatchVoted`, `BatchFinalized`, `AttestationClaimed`, etc.) already mark `address` parameters as `indexed`.

## `solc_remaps`

Mirror the Foundry `remappings.txt` so Slither resolves `@openzeppelin/*` imports without us reformatting source files.

## Re-evaluating exclusions

When upgrading Slither itself, re-read the detector catalogue and confirm each exclusion is still justified. The list is small on purpose — every entry is a deliberate trade-off, not a "make warnings stop" reflex.
