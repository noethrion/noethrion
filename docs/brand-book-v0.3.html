Mission & Doctrine

The visual identity exists to serve the protocol. Every design choice in this book traces back to a strategic pillar.

  • Gravitational, not imperial. The Foundation administrates an open standard; it does not own the ecosystem.
  • A physics law for energy economy. Noethrion is the verification primitive for generation, expressed through code and hardware.
  • Does not threaten sovereign currencies. Noethrion serves USD / EUR / CNY as an energy infrastructure layer.
  • 100% algorithmic emission of NOET (rule-based, no discretionary issuance); Foundation receives max 5% treasury with 4-year vesting.
  • Distribution via certified integration partners. The Foundation administers the open standard; partners build the products that use it. Enterprise relationships are mediated through certified third parties.
  • Multi-client implementation — 3–5 independent reference clients for resilience.

Baseline Rule (Formal)

The horizontal baseline below η is a defining brand element. v0.3 formalizes it.

Semantic meaning

The baseline represents:

  • Physical grounding — the protocol is anchored to physical electricity generation, not abstract claims.
  • The horizon line — energy is generated from sources that exist in physical reality.
  • A signature line — like a contract, every kWh is signed at a specific point in time.

Geometric rules (from Addendum §3.2.1)

When the baseline is present, it must be:

  • Horizontal (no diagonals, curves, or breaks except in approved variants like B3 wave or B4 pulse).
  • Centered horizontally on the η character.
  • Width: 50–65% of the visual width of η at the same scale (default 55%).
  • Positioned directly below the descender of η (the right-side descender).
  • Stroke weight: 0.6–1.5% of the height of η (default 0.7%).

D2 implementation (1024 × 1024 canvas, η in fallback Times New Roman Italic)

η visual width measured at ~307px on a 1024 canvas (font-size 720). 55% of that = 168px baseline span, centered on the visual ink center of η.

x1, x2: 423 → 591 (span 168px = 55% of η visual width)
y: 755 (just below η descender)
stroke: 4px, round linecap
color: #F5F1E8 (Cream / Bone)

Note: Instrument Serif Italic does not include Greek glyphs — η renders via the fallback chain (Times New Roman Italic on macOS/iOS, Georgia Italic on some Android). Measurements above are taken from Times New Roman Italic. Visual width in Georgia Italic is ~14% wider; this falls inside the 50–65% addendum range and does not require canvas-specific adjustment.

Logo DON'Ts

These are formally prohibited. Violations break brand integrity and must be rejected at design review.

✗Don't rotate the η. It is upright italic, never angled differently.
✗Don't change colors. The current palette is locked for consistency, even if not yet validated.
✗Don't add effects. No drop shadows, glows, embosses, bevels, gradients, or 3D treatments.
✗Don't crop the η. The full character must remain visible.
✗Don't break the baseline rule (geometry per Section 7).
✗Don't stretch. Aspect ratio is always preserved — no horizontal-only or vertical-only scaling.
✗Don't place on busy backgrounds. Minimum 0.5× η-height clearspace on all sides.
✗Don't combine η with other letters in a "wordmark style" (only the approved E2 mono lockup).
✗Don't use η as a bullet point or decorative element in body text. It is a logo, not a glyph.
✗Don't put η inside circles or shapes (except the approved G5 Foundation Seal).
✗Don't change the wordmark font. "NOETHRION" is set in Inter Tight Light (300) with letter-spacing.
✗Don't use the "Initial Development Co." name in branded materials. Always say "Noethrion".
✗Don't modify provided SVGs without explicit permission from brand stewardship.
✗Don't create new variants outside the brand stewardship process.

Validation Testing Protocol

Before locking the primary mark choice, run a structured visual test with at least 5 reviewers. The goal: confirm D2 actually communicates what we think it does, before it's locked into surfaces that are hard to revise later.

Step 1 — solo recognition. Show D2 alone, no context, and ask: "What does this make you think of?" Capture verbatim responses. Don't lead.

Step 2 — comparative match. Show D2 alongside 2–3 alternatives (D1, D3, D4). Ask: "Which feels more like X?" — where X cycles through: seriousness, technology, energy, infrastructure.

Step 3 — record & aggregate. Note feedback patterns. Look for clustering vs. disagreement.

Step 4 — decide.

  1. If N ≥ 5 results land consistently positive → lock D2.
  2. If split or negative → pivot to the next candidate (D1, D3, D4) and re-test.
  3. If no candidate clears the bar → iterate the design, do not force a locked choice.

Until validation completes, treat the working primary as a default — not a commitment.
